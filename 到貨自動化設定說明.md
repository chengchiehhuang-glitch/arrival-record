# 到貨管理系統 — 自動化設定說明

## 系統架構

```
手機 App（到貨登記）
      ↓ 儲存
Google Sheets（雲端資料庫）
      ↓ 每小時（Apps Script 觸發器）       ↓ 每天 18:00（Cowork 排程）
  checkArrivalNotify()              arrival_daily_report.py
  有新到貨 → 即時郵件（MailApp）      生成 Excel 日報 → 郵件附件
```

---

## 一、即時到貨通知（Google Apps Script）

即時通知由 Google Apps Script 直接在 Google 內部執行，不經過 Cowork，穩定度最高。

### 設定步驟

**步驟 1：打開 Apps Script 編輯器**

打開到貨登記的 Google Sheet → 上方選單「擴充功能」→「Apps Script」

**步驟 2：新增檔案並貼上程式碼**

在 Apps Script 編輯器左側，點「+」→「指令碼」→ 取名為 `arrival_notify`。
把 `arrival_notify_gas.js` 的內容整段貼進去。

貼好後確認設定區的值：

| 設定項目 | 說明 | 預設值 |
|---------|------|--------|
| `sheetName` | 工作表 tab 名稱 | `"到貨記錄"` |
| `mailTo` | 收件人信箱 | `"chengchieh.huang@gmail.com"` |
| `mailCc` | 副本信箱（留空不寄） | `""` |
| `quietStart` | 靜默開始（24h 格式） | `20`（晚上 8 點） |
| `quietEnd` | 靜默結束（24h 格式） | `7`（早上 7 點） |

**步驟 3：建立觸發器（一鍵完成）**

在編輯器上方工具列，找到函式下拉選單（在儲存按鈕旁邊），選 `setupArrivalNotifyTrigger` → 按「▶ 執行」。

第一次執行會要求授權，點「允許」即可。執行完畢後，每小時的自動觸發器就建好了。

不需要按「部署」，觸發器是綁在專案上的，跟對外部署無關。

**驗證觸發器：** 點左側選單「觸發條件」（鬧鐘圖示 ⏰），應看到一筆 `checkArrivalNotify` 每小時觸發器。

### 手動測試

選函式 `testArrivalNotify` → 按「▶ 執行」。這會模擬首次執行並寄出一封通知信。確認能收到信就代表設定成功。

### 通知邏輯說明

| 功能 | 說明 |
|------|------|
| 新到貨偵測 | 比對上次執行時間（存在 Script Properties），只通知新增的 |
| 靜默時段 | 20:00～07:00 不發通知（避免打擾） |
| 狀態儲存 | `PropertiesService.getScriptProperties()` 的 `notify_last_seen` |
| 無到貨 | 無新到貨時安靜略過，不發空信 |
| 郵件發送 | 使用 `MailApp.sendEmail()`，不需要 SMTP 密碼，直接用 Google 帳號發信 |

## 二、每日到貨日報（Cowork 排程）

日報由 Cowork 排程在每天 18:00 執行 Python 腳本，產生 Excel 日報並寄送郵件。

### 設定步驟

**步驟 1：放置腳本**

將檔案複製到桌面的 Claude 資料夾：

```
C:\Users\你的帳號\Desktop\Claude\
  └── arrival_daily_report.py   ← 日報腳本
```

**步驟 2：填入設定值**

用記事本開啟 .py 檔，找到「設定區」填入：

| 設定項目 | 說明 |
|---------|------|
| `GAS_URL` | 到貨登記 HTML 裡已設定的 Apps Script URL |
| `SMTP_USER` | 你的 Gmail 信箱（作為寄件人） |
| `SMTP_PASS` | Gmail **應用程式密碼**（非登入密碼，見下方說明） |
| `MAIL_TO` | 主管的信箱 |
| `MAIL_CC` | 副本信箱（可留空 `""`） |
| `REPORT_DIR` | 日報 Excel 儲存位置（預設桌面 Claude 資料夾） |

### 取得 Gmail 應用程式密碼

1. 前往 myaccount.google.com → 安全性
2. 開啟「兩步驟驗證」（需先啟用）
3. 搜尋「應用程式密碼」→ 新增 → 選「郵件」
4. 複製產生的 16 碼密碼填入 `SMTP_PASS`

**步驟 3：安裝 Python 套件**

在 Cowork 對話視窗輸入：

> 幫我安裝 arrival_daily_report.py 需要的套件

Cowork 會自動執行：
```
python -m pip install openpyxl requests
```

**步驟 4：Cowork 排程**

排程已建立，設定如下：

- **名稱**：`arrival-daily-report`
- **排程**：每天 18:00（`0 18 * * *`）

## 三、常見問題

| 問題 | 解決方法 |
|------|---------|
| 即時通知沒收到 | 到 Apps Script →「觸發條件」確認觸發器存在；選 `testArrivalNotify` 手動測試 |
| Apps Script 授權失敗 | 點「進階」→「前往（不安全）」→「允許」 |
| 找不到工作表 | 確認 `sheetName` 與 Sheet tab 名稱完全一致 |
| 日報郵件發送失敗 | 確認 SMTP_PASS 是「應用程式密碼」而非登入密碼 |
| 日報抓不到 Sheets 資料 | 確認 GAS_URL 正確，且 Apps Script 已部署為「所有人可存取」 |
| openpyxl 未安裝 | 執行 `python -m pip install openpyxl` |
